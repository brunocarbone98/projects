01
DEVSECOPS · SECURITY MONITORING

CyberShield

A microservices backend with security built into every stage of the development cycle, from the code to the production infrastructure.

What it is

An automated microservices backend built on a CI/CD pipeline that runs static code analysis and API tests on every change. The infrastructure is containerized and scalable, with real-time intrusion detection through alerts and centralized logging of all system activity.

Purpose

To demonstrate DevSecOps practices: integrating security and code quality into the development flow itself, rather than treating them as a final step. The system automatically validates every change before deploying it and monitors the production infrastructure, reflecting the ability to build software that is secure, observable, and ready to scale.

Stack used
Java · Spring Boot
Microservices backend
GitHub Actions
CI/CD pipeline orchestrating build, analysis, and tests
SonarQube
Static code analysis: quality and vulnerabilities
REST Assured
Automated API testing
Docker
Containerization of the services
AWS
Scalable infrastructure
Splunk
Centralized logging and real-time intrusion detection
02
FULL-STACK · DISTRIBUTED ARCHITECTURE

Full-Stack Hub

A reactive, multilingual interface on top of a distributed architecture that connects multiple languages and services into a single system.

See repository ↗
What it is

A reactive, SEO-optimized web interface backed by a distributed architecture: a transactional Node.js API connected to a PostgreSQL database, alongside microservices written in Python. The frontend and backend work together as a single hub with multilingual support.

Purpose

To demonstrate end-to-end full-stack capability: designing a modern, fast frontend while orchestrating a distributed backend where different languages and services (Node.js and Python) communicate and share persistence. It reflects experience with real production architectures, not just a static site.

Data persistence

FRONTEND · VANILLA WEB

Personal Web Portfolio

This very site: a two-page portfolio built from scratch without frameworks — pure web technologies only.

What it is

A personal portfolio made up of two connected pages: the main page — with my introduction, experience, and skills — and this projects page. Everything is built from scratch with plain HTML, CSS, and JavaScript, and includes a hero greeting that rotates through nine languages, reveal and typing animations, a background that subtly shifts colors, and a fully responsive design.

Purpose

To demonstrate command of frontend fundamentals without relying on frameworks: building an interactive, polished experience using only native web technologies. It is also my calling card in action: the project you are evaluating is the very page you are reading it on.
